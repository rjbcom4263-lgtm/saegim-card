### Step5 — gt.html 리빌딩 (이전 세션)

기존 isometric canvas 방식 완전 폐기 → 새 구조로 교체.

| 항목 | 내용 |
|------|------|
| 레이아웃 | 5×5 CSS 그리드 (이소메트릭 → 플랫) |
| 저장 방식 | localStorage (QR 코드별 분리: `gardentag_v1_1_{CODE}`) |
| 탭 구성 | 정원 / 인벤토리 / 도감 / 마이홈 / 관광지 / 상점 |
| 성장 시스템 | watered_until + grown_seconds 기반 시간 성장 |
| 아이템 ID | item_id 기반 (개수 카운트 아님, 개별 인스턴스) |
| Firestore 구조 | `buildFirestorePreview()`로 경로 확정 |

확정된 Firestore 경로:
```
garden_tags/{code}                    ← 메인 문서
garden_tags/{code}/plots/{plotId}     ← 정원 칸
garden_tags/{code}/codex/{resultId}   ← 도감
garden_tags/{code}/home_slots/{slotId}← 마이홈 슬롯
garden_tags/{code}/visits/{key}       ← 방문 기록
garden_items/{itemId}                 ← 개별 아이템
```

### Step6A — gt.html Firebase 실제 연결 (오늘)

`storageMode: "local"` → `"firebase"` 전환, 실제 Firestore 연동.

**변경 파일:** `gt.html`

변경 내용:

1. **APP_CONFIG 업데이트**
   - `version: "V1.1-step6a"`
   - `storageMode: "firebase"`
   - Firebase credentials 입력 (saegim-memory 프로젝트)

2. **Firebase SDK 추가** (`<head>` 안)
   ```html
   <script src="https://www.gstatic.com/firebasejs/10.14.1/firebase-app-compat.js"></script>
   <script src="https://www.gstatic.com/firebasejs/10.14.1/firebase-firestore-compat.js"></script>
   ```

3. **initFirebase()** — 페이지 load() 시 자동 호출, `gtDb` 초기화

4. **saveToFirebase()** — `buildFirestorePreview()` 결과를 batch.set으로 Firestore 저장

5. **loadFromFirebase()** — 다음 경로에서 state 재구성:
   - `garden_tags/{code}` (메인)
   - `/plots`, `/codex`, `/home_slots`, `/visits` (서브컬렉션)
   - `garden_items` where `owner_id == USER_CODE`

6. **load() async 전환** — Firebase 로드 먼저 시도 → 없으면 localStorage fallback

7. **save() 업데이트** — localStorage 저장 + Firebase saveToFirebase() 동시 호출

8. **init 시퀀스** — `(async()=>{ await load(); renderAll(); setInterval(...); })()`

9. **저장 방식 표시** — 연결 성공 시 상단에 `🔥 Firebase` 표시

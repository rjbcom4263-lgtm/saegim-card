### git lock 파일 문제

**증상:**
```
fatal: cannot lock ref 'HEAD': Unable to create '.git/HEAD.lock': File exists.
fatal: Unable to create '.git/index.lock': File exists.
```

**원인:**  
샌드박스(Linux)에서 git 명령 실행 시 Windows 마운트 폴더에 `.lock` 파일 생성됨.  
git이 권한 문제나 인터럽트로 비정상 종료되면 lock 파일이 남아 다음 git 작업을 막음.

**해결:**  
`deploy.bat` 파일로 lock 파일 자동 삭제 후 커밋/배포:

```bat
@echo off
del /f /q ".git\index.lock" 2>nul
del /f /q ".git\HEAD.lock" 2>nul
del /f /q ".git\REBASE_HEAD.lock" 2>nul
del /f /q ".git\MERGE_HEAD.lock" 2>nul
del /f /q ".git\packed-refs.lock" 2>nul
git add -A
git commit -m "%~1"
git push
firebase deploy --only hosting
echo.
echo Done!
```

**사용법:**
```
cd C:\Users\새김\saegim-card
deploy.bat "커밋 메시지"
```

### .bat 파일 인코딩 문제

**증상:**
```
'ul'은(는) 내부 또는 외부 명령이 아닙니다.
'ock'은(는) 내부 또는 외부 명령이 아닙니다.
```

**원인:**  
배치 파일이 LF(Unix) 줄바꿈으로 저장됨.  
Windows CMD는 CRLF(`\r\n`) 줄바꿈만 정상 인식.  
LF로 저장된 .bat 파일은 명령어가 잘리거나 합쳐져서 오작동.

**해결:**  
Python으로 CRLF 바이트 직접 지정해서 저장:
```python
with open('deploy.bat', 'wb') as f:
    f.write(content.encode('ascii'))
# content 내 줄바꿈을 \r\n으로 작성
```

확인 방법 (xxd로 0d 0a 확인):
```
xxd deploy.bat | head -5
# 00000000: 4065 6368 6f20 6f66 660d 0a ...
#                                  ^^^^^ CRLF 정상
```

### PDF 저장 오류 — Invalid string length

**증상:**
```
PDF 저장 오류: Invalid string length
```

**원인:**  
600 DPI A4 캔버스 크기 = 4961×7016px.  
`canvas.toDataURL('image/png')` → base64 문자열 50MB+  
jsPDF 내부 문자열 길이 한계 초과.

**해결:**  
PNG → JPEG 0.92 품질로 변경:
```javascript
pdf.addImage(canvas.toDataURL('image/jpeg', 0.92), 'JPEG', 0, 0, 210, 297);
```
인쇄 품질 충분히 유지하면서 용량 10배 이상 감소.
